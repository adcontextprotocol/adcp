## Key terms

| Term | Definition |
|------|-----------|
| **Working group** | Contributors who develop specific parts of the AdCP specification |
| **RFC** | Request for Comments — the process for proposing specification changes |
| **Semantic versioning** | The versioning scheme (major.minor.patch) used for AdCP releases |
| **OpenRTB** | The existing real-time bidding protocol — AdCP complements rather than replaces it |
| **IAB taxonomies** | Industry-standard category and classification systems that AdCP builds on |
| **adagents.json** | Publisher-hosted agent capability declaration |
| **ads.cert** | Cryptographic verification standard for ad supply chain integrity |

## What you'll discuss with Addie

- How does a new feature get added to the AdCP specification?
- If a publisher already uses OpenRTB, how does AdCP fit alongside it?
