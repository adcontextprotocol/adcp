# get_plan_audit_logs

Retrieve governance state and audit trail for a plan. Returns budget tracking, validation history, and compliance summary.

## Request

```json
{
  "tool": "get_plan_audit_logs",
  "arguments": {
    "plan_id": "plan_q1_2026_launch",
    "buyer_campaign_ref": "q1-2026-spring-launch",
    "include_entries": true
  }
}
```

Omit `buyer_campaign_ref` to get aggregate state across all campaigns in the plan.

## Fields

### Request

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `plan_id` | string | Yes | The plan to retrieve. |
| `buyer_campaign_ref` | string | No | Filter to a specific campaign. Omit for plan-level aggregate. |
| `include_entries` | boolean | No | Include the full audit trail. Default: `false`. |

### Response

| Field | Type | Description |
|-------|------|-------------|
| `plan_id` | string | Plan identifier. |
| `plan_version` | number | Current plan version. |
| `status` | enum | `active`, `suspended`, or `completed`. |
| `budget` | object | Budget state. |
| `budget.authorized` | number | Total authorized budget from the plan. |
| `budget.committed` | number | Total budget committed from confirmed outcomes. |
| `budget.remaining` | number | Authorized minus committed. |
| `budget.utilization_pct` | number | Committed as a percentage of authorized. |
| `channel_allocation` | object | Current channel mix. Keyed by channel ID. |
| `channel_allocation[channel].committed` | number | Budget committed to this channel. |
| `channel_allocation[channel].pct` | number | Channel's share of the authorized total budget. |
| `campaigns` | array | Per-campaign breakdown. |
| `campaigns[].buyer_campaign_ref` | string | Campaign identifier. |
| `campaigns[].status` | enum | `active`, `suspended`, or `completed`. |
| `campaigns[].committed` | number | Budget committed in this campaign. |
| `campaigns[].active_media_buys` | array | Media buy IDs currently active. |
| `summary` | object | Aggregate validation and outcome statistics. |
| `summary.checks_performed` | number | Total governance checks performed. |
| `summary.outcomes_reported` | number | Total outcomes reported. |
| `summary.statuses` | object | Count of each governance check status (`approved`, `denied`, `conditions`, `escalated`). |
| `summary.findings_count` | number | Total findings from `report_plan_outcome`. |
| `summary.escalations` | array | All escalations and their resolutions. |
| `summary.escalations[].check_id` | string | The escalated governance check. |
| `summary.escalations[].reason` | string | Why it was escalated. |
| `summary.escalations[].resolution` | string | How it was resolved (e.g., `approved_by_human`, `rejected_by_human`). |
| `summary.escalations[].resolved_at` | string | ISO 8601 resolution timestamp. |
| `summary.drift_metrics` | object | Aggregate governance metrics for detecting oversight drift. See [specification](/docs/governance/campaign/specification#drift-detection). |
| `summary.drift_metrics.escalation_rate` | number | Fraction of checks that resulted in escalation (0-1). |
| `summary.drift_metrics.escalation_rate_trend` | enum | `increasing`, `stable`, or `declining`. |
| `summary.drift_metrics.auto_approval_rate` | number | Fraction of checks approved without human intervention (0-1). |
| `summary.drift_metrics.human_override_rate` | number | Fraction of escalations where the human overrode the agent (0-1). |
| `summary.drift_metrics.mean_confidence` | number | Average confidence score across findings (0-1). Present when findings include confidence. |
| `summary.drift_metrics.thresholds` | object | Organization-defined thresholds for drift metrics. When a metric crosses its threshold, the governance agent includes a finding. |
| `summary.drift_metrics.thresholds.escalation_rate_max` | number | Maximum acceptable escalation rate. |
| `summary.drift_metrics.thresholds.escalation_rate_min` | number | Minimum acceptable escalation rate. A rate below this may indicate eroding oversight. |
| `summary.drift_metrics.thresholds.auto_approval_rate_max` | number | Maximum acceptable auto-approval rate. |
| `summary.drift_metrics.thresholds.human_override_rate_max` | number | Maximum acceptable human override rate. |
| `entries` | array | Ordered audit trail (only when `include_entries` is `true`). |
| `entries[].id` | string | Entry identifier. |
| `entries[].type` | enum | `check` or `outcome`. |
| `entries[].timestamp` | string | ISO 8601 timestamp. |
| `entries[].tool` | string | The AdCP tool (present for `check` entries). |
| `entries[].status` | enum | Governance check status (present for `check` entries). |
| `entries[].binding` | enum | `proposed` or `committed` (present for `check` entries). |
| `entries[].outcome` | enum | Outcome type (present for `outcome` entries). |
| `entries[].committed_budget` | number | Budget committed (present for `completed` outcome entries). |
| `entries[].media_buy_id` | string | Media buy ID (present for `delivery` outcome entries). |
| `entries[].outcome_status` | string | Outcome status (present for `outcome` entries). |

## Error codes

| Code | Recovery | Description |
|------|----------|-------------|
| `PLAN_NOT_FOUND` | correctable | No plan with this ID. |
