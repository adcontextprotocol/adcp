# validate_plan_action

Validate an orchestrator action against a campaign plan before it is sent to a seller. This is the "before" half of the governance loop.

## Request

```json
{
  "tool": "validate_plan_action",
  "arguments": {
    "plan_id": "plan_q1_2026_launch",
    "operator": "https://buyer-orchestrator.example.com",
    "buyer_campaign_ref": "q1-2026-spring-launch",
    "tool": "create_media_buy",
    "payload": {
      "buyer_ref": "q1-launch-pinnacle-001",
      "account": { "agent_url": "https://seller.example.com", "id": "acc_123" },
      "brand": { "domain": "acmecorp.com" },
      "start_time": "2026-03-15T00:00:00Z",
      "end_time": "2026-06-15T00:00:00Z",
      "packages": [
        {
          "product_id": "premium_video_300k",
          "budget": 150000,
          "pricing_option_id": "cpm_guaranteed",
          "targeting_overlay": {
            "geo": { "include": [{ "type": "country", "code": "US" }] }
          }
        }
      ]
    }
  }
}
```

## How it works

The governance agent evaluates the action against the plan synced via [`sync_plans`](./sync_plans). It infers the lifecycle phase from the `tool` being validated:

| tool | Phase |
|------|-------|
| `get_products` | Discovery -- validates search intent, seller eligibility, product suitability |
| `create_media_buy` | Purchase -- validates budget authority, targeting compliance, flight dates |
| `update_media_buy` | Purchase -- validates change magnitude, reallocation thresholds |

Phase context is cumulative. During **purchase**, the governance agent considers what was discovered during **discovery**.

The `action_id` in the response is used by [`report_plan_outcome`](./report_plan_outcome) to link the seller's response back to this validated action.

## Verdicts

### approved

All checks passed. The orchestrator may proceed.

### rejected

```json
{
  "action_id": "act_rej_001",
  "verdict": "rejected",
  "categories": [
    {
      "category_id": "budget_authority",
      "status": "failed",
      "explanation": "This action would commit $600,000 total, exceeding the authorized budget of $500,000.",
      "details": {
        "total_committed": 450000,
        "this_action": 150000,
        "authorized_total": 500000
      }
    }
  ]
}
```

The orchestrator MUST NOT execute the action.

### modification_required

```json
{
  "action_id": "act_mod_001",
  "verdict": "modification_required",
  "categories": [
    {
      "category_id": "brand_policy",
      "status": "failed",
      "explanation": "Brand compliance policy (age_gating_18_plus) prohibits targeting users under 18. The targeting overlay does not include an age restriction.",
      "details": {
        "policy_id": "age_gating_18_plus",
        "policy_source": "registry"
      }
    }
  ],
  "modifications": [
    {
      "field": "packages[0].targeting_overlay.demographics.age",
      "current_value": null,
      "suggested_value": { "min": 18 },
      "reason": "Brand compliance: age restriction policy requires minimum age 18."
    }
  ]
}
```

The orchestrator should apply the suggested modifications and re-submit.

### escalated

```json
{
  "action_id": "act_esc_001",
  "verdict": "escalated",
  "categories": [
    {
      "category_id": "budget_authority",
      "status": "warning",
      "explanation": "This $45,000 reallocation from display to video exceeds the $25,000 reallocation threshold."
    }
  ],
  "escalation": {
    "reason": "Budget reallocation exceeds the authorized threshold.",
    "severity": "critical",
    "requires_human": true
  }
}
```

Escalation severity levels:

- **`info`** -- Logged for audit, no action required
- **`warning`** -- Agent may proceed but human should review
- **`critical`** -- Agent MUST NOT proceed until human approves

## Fields

### Request

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `plan_id` | string | Yes | The plan to validate against. |
| `operator` | string | Yes | URL of the agent making the request. |
| `buyer_campaign_ref` | string | Yes | Buyer's campaign identifier. The governance agent tracks state per campaign within a plan. |
| `tool` | string | Yes | The AdCP tool being validated (e.g., `create_media_buy`, `get_products`). |
| `payload` | object | Yes | The full tool arguments, exactly as they would be sent to the seller. |

### Response

| Field | Type | Description |
|-------|------|-------------|
| `action_id` | string | Unique identifier for this validation record. Use in `report_plan_outcome` to link outcomes. |
| `verdict` | enum | `approved`, `rejected`, `modification_required`, or `escalated`. |
| `categories` | array | Per-category evaluation results. |
| `categories[].category_id` | string | Which validation category. |
| `categories[].status` | enum | `passed`, `failed`, or `warning`. |
| `categories[].explanation` | string | Human-readable explanation. |
| `categories[].details` | object | Optional structured details. |
| `modifications` | array | Present only when verdict is `modification_required`. |
| `escalation` | object | Present only when verdict is `escalated`. |
| `escalation.reason` | string | Human-readable explanation of why the action was escalated. |
| `escalation.severity` | enum | `info`, `warning`, or `critical`. |
| `escalation.requires_human` | boolean | Whether human approval is required. |

## Error codes

| Code | Recovery | Description |
|------|----------|-------------|
| `PLAN_NOT_FOUND` | correctable | No plan with this ID. Sync it first. |
| `CAMPAIGN_SUSPENDED` | correctable | Campaign governance is suspended pending human review. |
| `UNSUPPORTED_TOOL` | correctable | The governance agent does not validate this tool type. |
