Understanding how the pieces fit together helps when navigating AdCP resources.

## Organizations

| Organization | Purpose | URL |
|--------------|---------|-----|
| **AgenticAdvertising.org** | Membership community, working groups, member profiles | [agenticadvertising.org](https://agenticadvertising.org) |
| **AdCP (Ad Context Protocol)** | Protocol specification, schemas, documentation | [adcontextprotocol.org](https://adcontextprotocol.org) |

**Key distinction**: AgenticAdvertising.org is the *community*. AdCP is the *protocol*. Members join AgenticAdvertising.org to develop and adopt AdCP.

## Component Architecture

```
┌─────────────────────────────────────────────────────────────┐
│                     PRINCIPALS                              │
│            (Advertisers, Agencies, Brands)                  │
└─────────────────────┬───────────────────────────────────────┘
                      │ Authenticate via
                      ▼
┌─────────────────────────────────────────────────────────────┐
│                  ORCHESTRATING AGENTS                       │
│         (AI Agents executing media buying tasks)            │
└─────────────────────┬───────────────────────────────────────┘
                      │ AdCP Tasks (MCP/A2A)
                      ▼
┌─────────────────────────────────────────────────────────────┐
│                    SALES AGENTS                             │
│       (Publisher-side agents handling inventory)            │
└─────────────────────┬───────────────────────────────────────┘
                      │ Platform APIs
                      ▼
┌─────────────────────────────────────────────────────────────┐
│                 AD PLATFORMS/SERVERS                        │
│    (Google Ad Manager, TTD, Meta, etc.)                     │
└─────────────────────────────────────────────────────────────┘
```

### Principals

The humans or organizations with authority to spend money:
- Advertisers
- Agencies
- Brands

Principals authenticate and authorize orchestrating agents to act on their behalf.

### Orchestrating Agents

AI agents that execute media buying workflows:
- Receive instructions from principals
- Call AdCP tasks on sales agents
- Handle clarifications and approvals
- Report results back to principals

### Sales Agents

Server implementations that expose AdCP tasks:
- Translate AdCP to platform-specific APIs
- Handle inventory discovery and pricing
- Manage creative workflows
- Process media buy orders

### Ad Platforms

The underlying advertising systems:
- Google Ad Manager
- The Trade Desk
- Meta Business Manager
- Amazon Ads
- And more...

## Core AdCP Tasks

| Task | Purpose | Category |
|------|---------|----------|
| `get_products` | Discover inventory via natural language | Discovery |
| `list_creative_formats` | Get supported creative formats | Discovery |
| `create_media_buy` | Purchase advertising inventory | Execution |
| `get_media_buy` | Check campaign status | Execution |
| `update_media_buy` | Modify running campaigns | Execution |
| `sync_creatives` | Upload/register creative assets | Creative |
| `get_signals` | Find targeting signals | Signals |
| `activate_signal` | Enable signal for targeting | Signals |

## Repositories

All repos are under the `adcontextprotocol` GitHub organization.

| Repo | Purpose | Key Contents |
|------|---------|--------------|
| **adcp** | Protocol and AgenticAdvertising.org | JSON schemas, TypeScript/Python SDKs, protocol docs |
| **salesagent** | Reference sales agent implementation | Sales agent code, deployment configs |
| **creative-agent** | Reference creative agent | Creative workflows, format validation |

## Schema Locations

- **Base URL**: `https://adcontextprotocol.org/schemas/v2/`
- **Core schemas**: `/schemas/v2/core/` (product, media-buy, creative, error)
- **Task schemas**: `/schemas/v2/media-buy/` and `/schemas/v2/signals/`
- **Enum schemas**: `/schemas/v2/enums/`
- **Registry**: [`/schemas/v2/index.json`](https://adcontextprotocol.org/schemas/v2/index.json)
