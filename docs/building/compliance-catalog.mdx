Every AdCP agent declares its `domains` and `specialisms` in `get_adcp_capabilities`. Each declaration maps to a compliance bundle at `/compliance/{version}/` that the storyboard runner executes to verify the claim.

This page is the human-readable index of that taxonomy. The machine-readable equivalent is `/compliance/{version}/index.json`.

## Domains

Broad agent categories. An agent claims a domain by listing it in `domains` and must pass the domain's baseline storyboard plus every [universal](/docs/building/validate-your-agent#storyboard-taxonomy) storyboard.

| Domain | Purpose | Baseline path |
|--------|---------|---------------|
| `media-buy` | Campaign creation, package management, delivery optimization, conversion tracking | `domains/media-buy/` |
| `creative` | Creative asset management, format discovery, rendering | `domains/creative/` |
| `signals` | Audience signal discovery and activation | `domains/signals/` |
| `governance` | Property governance, brand standards, compliance | `domains/governance/` |
| `brand` | Brand identity, rights discovery, rights acquisition | `domains/brand/` |
| `sponsored-intelligence` | AI-mediated commerce and conversational sponsored experiences | `domains/sponsored-intelligence/` |

<Tip>
An agent can claim multiple domains — a full-stack media-buy platform might claim `media-buy`, `creative`, and `signals`. The runner executes all matching baselines.
</Tip>

## Specialisms

Specific capability claims. Each specialism lives under exactly one domain. An agent claiming a specialism must pass the specialism's storyboard in addition to its parent domain baseline.

### media-buy

| Specialism | Purpose |
|-----------|---------|
| `sales-guaranteed` | Guaranteed media buys with human IO approval |
| `sales-non-guaranteed` | Non-guaranteed auction-based media buys |
| `sales-proposal-mode` | Media buys negotiated via proposal acceptance |
| `sales-catalog-driven` | Catalog-driven commerce with conversion tracking |
| `sales-broadcast-tv` | Broadcast linear TV with guaranteed inventory and FCC cancellation rules |
| `sales-streaming-tv` | Connected TV / streaming with ad-supported video inventory |
| `sales-social` | Social media advertising platform with self-service flows |
| `sales-exchange` | Programmatic exchange or SSP exposing auction-based inventory |
| `sales-retail-media` | Retail media network seller — onsite, offsite, in-store inventory |

### creative

| Specialism | Purpose |
|-----------|---------|
| `creative-ad-server` | Creative ad server with tag-based delivery |
| `creative-generative` | Generative creative agent producing assets on demand |
| `creative-template` | Creative template and transformation agent |

### signals

| Specialism | Purpose |
|-----------|---------|
| `signal-owned` | Owned signal agent exposing first-party segments |
| `signal-marketplace` | Marketplace signal agent reselling third-party data |

### governance

| Specialism | Purpose |
|-----------|---------|
| `audience-sync` | Syncs buyer-provided audience segments into a platform for activation |
| `content-standards` | Content standards enforcement (brand safety, policy compliance) |
| `inventory-lists` | Property and collection list governance — curated inventory groupings for targeting and authorization |
| `governance-delivery-monitor` | Campaign delivery monitoring with drift detection |
| `governance-spend-authority` | Conditional spend approval and human-in-the-loop governance |
| `measurement-verification` | Third-party measurement and verification across domains (viewability, brand safety, attribution, SI outcomes) |

### brand

| Specialism | Purpose |
|-----------|---------|
| `brand-rights` | Brand identity and rights licensing (talent, music, stock media) |

## How to claim

Declare your domains and specialisms in `get_adcp_capabilities`:

```json
{
  "domains": ["media-buy", "creative"],
  "specialisms": ["sales-guaranteed", "creative-template"]
}
```

The storyboard runner:

1. Runs every storyboard in `/compliance/{version}/universal/`
2. Runs each claimed domain's baseline at `/compliance/{version}/domains/{domain}/`
3. Runs each claimed specialism's storyboard at `/compliance/{version}/specialisms/{id}/`

If any storyboard fails, your agent is not compliant for that claim. See [Validate Your Agent](/docs/building/validate-your-agent) for how to run the suite locally.

## Source of truth

The machine index is published alongside schemas:

| Path | Contents |
|------|----------|
| `/compliance/{version}/index.json` | Enumerated domains + specialisms + universal storyboards |
| `/schemas/{version}/enums/adcp-domain.json` | Domain enum used by `get_adcp_capabilities.domains` |
| `/schemas/{version}/enums/specialism.json` | Specialism enum used by `get_adcp_capabilities.specialisms` |

The build pipeline verifies these three artifacts stay in sync — adding a specialism YAML without updating the enum (or vice versa) fails the build.

<Note>
The catalog on this page is maintained by hand to give human context. The authoritative enumeration is always `/compliance/{version}/index.json`.
</Note>
