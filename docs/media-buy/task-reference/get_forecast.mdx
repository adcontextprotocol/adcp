Return a delivery forecast for a specific product configuration without requiring a full `get_products` call. Use for iterative campaign planning, budget optimization, and refreshing forecasts after initial discovery.

**Response Time**: ~60 seconds (may involve model computation)

**Request Schema**: [`/schemas/v1/media-buy/get-forecast-request.json`](https://adcontextprotocol.org/schemas/v1/media-buy/get-forecast-request.json)
**Response Schema**: [`/schemas/v1/media-buy/get-forecast-response.json`](https://adcontextprotocol.org/schemas/v1/media-buy/get-forecast-response.json)

## Request Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `product_id` | string | Yes | ID of the product to forecast, from a previous `get_products` response |
| `budget` | number | Yes | Budget to forecast delivery for, in the product's pricing currency |
| `start_date` | string | Yes | Campaign start date (YYYY-MM-DD) |
| `end_date` | string | Yes | Campaign end date (YYYY-MM-DD) |
| `targeting_overlay` | [Targeting](/docs/media-buy/task-reference/create_media_buy#targeting) | No | Targeting overlay to apply. Narrowing targeting reduces forecasted reach. |
| `optimization_goal` | [OptimizationGoal](/docs/media-buy/task-reference/create_media_buy#optimization) | No | Optimization goal. Some sellers provide goal-specific forecasts (e.g., estimated conversions at a given budget). |
| `pricing_option_id` | string | No | Pricing option to use. When omitted, the seller uses the product's default pricing option. |
| `account` | [AccountRef](/docs/building/integration/accounts-and-agents#account-references) | No | Account reference for sellers that require an account for forecasting |

## Response

### Success

| Field | Description |
|-------|-------------|
| `success` | `true` |
| `product_id` | Product ID this forecast is for (echoed from request) |
| `forecast` | [DeliveryForecast](/docs/media-buy/task-reference/get_products#delivery-forecast) object with `points`, `method`, `currency`, and optional `valid_until` |

### Error

| Field | Description |
|-------|-------------|
| `success` | `false` |
| `errors` | Array of [Error](/docs/building/implementation/error-handling) objects |

## Examples

### Basic forecast

```json
{
  "$schema": "https://adcontextprotocol.org/schemas/v1/media-buy/get-forecast-request.json",
  "product_id": "prod_premium_video",
  "budget": 50000,
  "start_date": "2026-04-01",
  "end_date": "2026-04-30"
}
```

### Forecast with targeting overlay

```json
{
  "$schema": "https://adcontextprotocol.org/schemas/v1/media-buy/get-forecast-request.json",
  "product_id": "prod_premium_video",
  "budget": 50000,
  "start_date": "2026-04-01",
  "end_date": "2026-04-30",
  "targeting_overlay": {
    "geo_countries": ["US"],
    "geo_regions": ["US-CA", "US-NY"]
  },
  "optimization_goal": {
    "kind": "metric",
    "metric": "completed_views"
  }
}
```

### Forecast with specific pricing option

```json
{
  "$schema": "https://adcontextprotocol.org/schemas/v1/media-buy/get-forecast-request.json",
  "product_id": "prod_display_run_of_site",
  "budget": 10000,
  "start_date": "2026-05-01",
  "end_date": "2026-05-31",
  "pricing_option_id": "po_cpm_guaranteed",
  "account": { "account_id": "acc_acme_pinnacle" }
}
```

## Workflow

`get_forecast` fits into the campaign planning loop:

1. **Discover products**: Use [`get_products`](/docs/media-buy/task-reference/get_products) to find relevant inventory
2. **Iterate on forecasts**: Use `get_forecast` to test different budgets, dates, and targeting overlays
3. **Create the buy**: Use [`create_media_buy`](/docs/media-buy/task-reference/create_media_buy) once satisfied with the forecast

This is more efficient than calling `get_products` repeatedly since `get_forecast` skips discovery and goes straight to forecast computation for a known product.

## Error Handling

| Error Code | Description | Resolution |
|------------|-------------|------------|
| `PRODUCT_UNAVAILABLE` | Product is sold out or no longer available | Choose a different product via `get_products` |
| `BUDGET_TOO_LOW` | Budget below seller's minimum | Increase budget or check `capabilities.media_buy.limits` |
| `UNSUPPORTED_FEATURE` | Targeting or goal not supported | Check `get_adcp_capabilities` and adjust request |
