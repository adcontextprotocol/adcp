This page enumerates every published AdCP version, its current status, and its end-of-life date. For the policy behind these states — release tiers, schema-change scope, cadence — see [Versioning & Governance](/docs/reference/versioning).

## At a glance

| Status | Versions | What it means |
|---|---|---|
| **Active** | 3.0 (current: 3.0.11) | Default for new integrations. Receives feature and security patches. |
| **Security-only** | 2.5.x (final: 2.5.3) | Unsupported on the AAO network. Security patches through **August 1, 2026 (UTC)**. Not safe for interoperable production — missing accounts and governance protections. |
| **End of life** | 2.0, 2.1 | Predate 2.5.x. No patches. Upgrade required. |
| **Planned** | 3.1 (target late June 2026), 3.2 (target late September 2026), 4.0 (target early 2027) | Not yet released. |

After August 1, 2026 (UTC), the entire 2.x line is fully deprecated. The 2.x schema URLs continue to resolve so existing integrations don't break silently, but those schemas receive no further updates. See [v2 Sunset](/docs/reference/v2-sunset) for the full timeline and migration paths.

2.x version numbers between published releases (2.2, 2.3, 2.4, 2.5.2, 2.6) appear only as in-flight changelog entries — they were either rolled forward into the next published release or deferred to 3.0. There is no implementation to integrate against.

## Common version questions

### What's "current"?
**3.0.11.** Pin your integration to `"3.0"` (release precision) and upgrade patches as they ship. See [Version negotiation](/docs/reference/versioning#version-negotiation).

### Why are there gaps in the 2.x numbering (no 2.2, 2.3, 2.4, 2.6)?
The 2.x line skipped version numbers during its working-group cycle. Some numbers appear as in-flight changelog entries but were never released — the changes were rolled forward into the next published version or deferred to 3.0. The five published 2.x releases are 2.0.0, 2.1.0, 2.5.0, 2.5.1, and 2.5.3.

### Can I still register a v2 agent?
Yes — v2 agents can be registered in the AAO registry but cannot be AAO Verified. Certification and verified-default discovery require v3.x. See [v2 Sunset → If you use the AAO registry](/docs/reference/v2-sunset#if-you-use-the-aao-registry).

### How long will 3.0 be supported?
At least 12 months after 4.0 GA. The previous-major support commitment is documented in [Versioning → Support window for previous major](/docs/reference/versioning#support-window-for-previous-major).

### Where is 4.0?
Targeted for early 2027. The current 3.x cycle is mid-stream — see the [planned releases table](/docs/reference/versioning#planned-releases).

## Choosing a version

| If you are… | Use |
|---|---|
| Building a new integration today | **3.0** (latest patch). |
| Running a 2.5.x integration | Plan to migrate to 3.0 before **Aug 1, 2026 (UTC)**. Start with the [migration guide](/docs/reference/migration). |
| Running a 2.0 or 2.1 integration | Upgrade now — these are out of support. |
| Trying an unreleased feature | See [prerelease upgrade notes](/docs/reference/migration/prerelease-upgrades) and [experimental status](/docs/reference/experimental-status). |
