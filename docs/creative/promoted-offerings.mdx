A traditional creative asset is a finished ad: one image, one video, one message. **Promoted Offerings** is different — it's a structured brief that carries everything a publisher needs to compose the right ad for each impression.

The buyer provides:

- **Brand identity** — who's advertising, their logos, colors, tone
- **Multiple offerings** — each a distinct campaign, product, or promotion with its own name, validity window, landing URL, and keywords
- **Per-offering creative pools** — each offering carries its own headlines, images, and videos; the publisher selects the best combination at serve time
- **Optional SI agent** — if the brand supports conversational experiences, any offering can escalate to a live chat with the brand agent

The key distinction from a simple asset pool: each offering is a *semantic unit*. It has a name, a validity window, keywords for intent matching, a destination URL, and its own creative assets. The publisher can serve the offering most relevant to a given user, not just pick from a flat list of files. When a user engages beyond the ad, the same offering can trigger an SI session.

This makes `promoted_offerings` the right choice when:

- You're running a universal format (PMax-style) where the publisher assembles the creative
- The brand has multiple promotions and you want the publisher to pick the most relevant one
- The brand supports conversational experiences alongside traditional ad serving
- You want creative performance optimization without managing hundreds of individual creative variants

For formats that don't use `promoted_offerings` as a creative asset, attach basic brand context to a campaign via the `brand` field on `create_media_buy`.

## How it works in a media buy

When a format lists a `promoted_offerings` asset in its definition, the buying agent submits the `PromotedOfferings` object as the value for that asset key in a creative manifest via `sync_creatives`:

```json
{
  "creatives": [
    {
      "creative_id": "summer-sale-universal",
      "name": "Summer Sale — Universal Format",
      "format_id": {
        "agent_url": "https://publisher.example/adcp",
        "id": "universal_multi_channel"
      },
      "assets": {
        "promoted_offerings": {
          "brand": { "domain": "acme.com" },
          "offerings": [
            {
              "offering_id": "summer-sale",
              "name": "Summer Sale",
              "landing_url": "https://acme.com/summer",
              "assets": [
                {
                  "asset_group_id": "headlines",
                  "asset_type": "text",
                  "items": [
                    {"content": "Shop the Summer Sale"},
                    {"content": "50% Off Everything"},
                    {"content": "Summer Savings Event"}
                  ]
                },
                {
                  "asset_group_id": "images_landscape",
                  "asset_type": "image",
                  "items": [
                    {"url": "https://cdn.acme.com/hero.jpg", "width": 1200, "height": 628}
                  ]
                }
              ]
            }
          ]
        }
      }
    }
  ]
}
```

The publisher picks which offering to show and selects the best headline/image/video combination from within each offering's asset groups.

## Buying platform workflow

### Step 1: Discover format requirements

Call `list_creative_formats` and look for formats with a `promoted_offerings` asset. The asset's `requirements` field tells you exactly what each offering must provide.

```json
{
  "assets": [
    {
      "item_type": "individual",
      "asset_id": "promoted_offerings",
      "asset_type": "promoted_offerings",
      "required": true,
      "requirements": {
        "requires": ["offerings", "offerings.assets", "offerings.landing_url", "brand.logos"],
        "offering_asset_constraints": [
          {
            "asset_group_id": "headlines",
            "asset_type": "text",
            "required": true,
            "min_count": 3,
            "max_count": 15,
            "asset_requirements": {"max_length": 30}
          },
          {
            "asset_group_id": "descriptions",
            "asset_type": "text",
            "required": true,
            "min_count": 2,
            "max_count": 5,
            "asset_requirements": {"max_length": 90}
          },
          {
            "asset_group_id": "images_landscape",
            "asset_type": "image",
            "required": true,
            "min_count": 1,
            "max_count": 20,
            "asset_requirements": {"aspect_ratio": "1.91:1", "min_width": 1200, "min_height": 628}
          },
          {
            "asset_group_id": "images_square",
            "asset_type": "image",
            "required": true,
            "min_count": 1,
            "max_count": 20,
            "asset_requirements": {"aspect_ratio": "1:1", "min_width": 600, "min_height": 600}
          }
        ]
      }
    }
  ]
}
```

### Step 2: Build conformant offerings

For each offering, provide asset groups matching the format's `offering_asset_constraints`. The `asset_group_id` values must match exactly what the format declares.

```json
{
  "offering_id": "summer-sale",
  "name": "Summer Sale",
  "landing_url": "https://acme.com/summer",
  "assets": [
    {
      "asset_group_id": "headlines",
      "asset_type": "text",
      "items": [
        {"content": "Shop the Summer Sale"},
        {"content": "50% Off Everything"},
        {"content": "Summer Savings Event"}
      ]
    },
    {
      "asset_group_id": "descriptions",
      "asset_type": "text",
      "items": [
        {"content": "Up to 50% off our entire summer collection. Limited time only."},
        {"content": "Shop now and save big on summer styles."}
      ]
    },
    {
      "asset_group_id": "images_landscape",
      "asset_type": "image",
      "items": [
        {"url": "https://cdn.acme.com/hero.jpg", "width": 1200, "height": 628},
        {"url": "https://cdn.acme.com/hero2.jpg", "width": 1200, "height": 628}
      ]
    },
    {
      "asset_group_id": "images_square",
      "asset_type": "image",
      "items": [
        {"url": "https://cdn.acme.com/sq.jpg", "width": 600, "height": 600}
      ]
    }
  ]
}
```

### Step 3: Submit via sync_creatives

Wrap the `PromotedOfferings` object as the value for the format's `promoted_offerings` asset key in the creative manifest. The publisher validates the asset groups against its constraints before accepting the creative.

### Conversational experiences

If the brand has an SI agent, add `si_agent_url` to the `PromotedOfferings` object. This enables the publisher to initiate SI sessions when users want to interact with an offering rather than just view an ad:

```json
{
  "brand": { "domain": "brand.com" },
  "si_agent_url": "https://agent.brand.com/mcp",
  "offerings": [...]
}
```

The `si_agent_url` and asset-pool creatives can coexist: the same offering can serve a display ad and also be available for a conversational experience.

## Schema Reference

### Promoted Offerings

**Schema URL**: [`/schemas/core/promoted-offerings.json`](https://adcontextprotocol.org/schemas/v2/core/promoted-offerings.json)

```typescript
interface PromotedOfferings {
  // Brand identity (required)
  brand: Brand;  // Brand reference with domain

  // Conversational AI endpoint (optional)
  si_agent_url?: string;  // MCP endpoint for SI sessions

  // What to promote (choose one or both)
  offerings?: Offering[];           // Explicit offering definitions
  product_selectors?: ProductSelectors;  // Select from brand catalog

  // Creative asset filtering (optional)
  asset_selectors?: AssetSelectors;
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `brand` | object | Yes | Brand reference with `domain` field |
| `si_agent_url` | string (uri) | No | MCP endpoint for conversational experiences |
| `offerings` | Offering[] | No | Array of specific offerings to promote |
| `product_selectors` | object | No | Selectors to choose products from brand catalog |
| `asset_selectors` | object | No | Filters to select specific creative assets |

### Offering

**Schema URL**: [`/schemas/core/offering.json`](https://adcontextprotocol.org/schemas/v2/core/offering.json)

An individual promotable item—a campaign, product, service, or promotion.

```typescript
interface Offering {
  // Identity (required)
  offering_id: string;   // Unique identifier
  name: string;          // Human-readable name

  // Description
  description?: string;
  tagline?: string;

  // Validity window
  valid_from?: string;  // ISO 8601 datetime
  valid_to?: string;    // ISO 8601 datetime

  // Destinations
  checkout_url?: string;  // Purchase/conversion URL
  landing_url?: string;   // Information page

  // Creative assets
  assets?: OfferingAssetGroup[];  // Structured asset groups
  portfolio_ref?: string;         // URL to creative portfolio

  // Discovery
  keywords?: string[];    // For intent matching
  categories?: string[];  // For categorization
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `offering_id` | string | Yes | Unique identifier for referencing in SI sessions |
| `name` | string | Yes | Human-readable offering name |
| `description` | string | No | Detailed description |
| `tagline` | string | No | Short promotional tagline |
| `valid_from` | datetime | No | When offering becomes available |
| `valid_to` | datetime | No | When offering expires |
| `checkout_url` | uri | No | URL for purchase flow |
| `landing_url` | uri | No | Landing page URL |
| `assets` | OfferingAssetGroup[] | No | Structured asset groups for this offering |
| `portfolio_ref` | uri | No | Reference to creative portfolio |
| `keywords` | string[] | No | Keywords for intent matching |
| `categories` | string[] | No | Categories for filtering |

### OfferingAssetGroup

**Schema URL**: [`/schemas/core/offering-asset-group.json`](https://adcontextprotocol.org/schemas/v2/core/offering-asset-group.json)

A typed pool of creative assets within an offering. Uses the same `asset_group_id` vocabulary as format-level asset definitions, enabling formats to declare per-group constraints on what each offering must provide.

```typescript
interface OfferingAssetGroup {
  asset_group_id: string;        // e.g., 'headlines', 'images_landscape', 'videos_portrait'
  asset_type: AssetContentType;  // Type of all items in this group
  items: Asset[];                // The assets (must match asset_type)
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `asset_group_id` | string | Yes | Matches format-level vocabulary (e.g., `headlines`, `descriptions`, `images_landscape`) |
| `asset_type` | AssetContentType | Yes | Content type of all items in this group |
| `items` | Asset[] | Yes | The assets; each item must match the declared `asset_type` |

### OfferingAssetConstraint

**Schema URL**: [`/schemas/core/requirements/offering-asset-constraint.json`](https://adcontextprotocol.org/schemas/v2/core/requirements/offering-asset-constraint.json)

Declared by a format to specify what asset groups each offering must provide and the technical requirements for items in each group. Used within `PromotedOfferingsAssetRequirements.offering_asset_constraints`.

```typescript
interface OfferingAssetConstraint {
  asset_group_id: string;         // Must match the offering's OfferingAssetGroup.asset_group_id
  asset_type: AssetContentType;   // Expected content type for this group
  required?: boolean;             // Whether the group must be present (default: true)
  min_count?: number;             // Minimum items required in the group
  max_count?: number;             // Maximum items allowed in the group
  asset_requirements?: AssetRequirements;  // Technical requirements per item
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `asset_group_id` | string | Yes | The group this constraint applies to (e.g., `headlines`, `images_landscape`) |
| `asset_type` | AssetContentType | Yes | Expected content type for items in this group |
| `required` | boolean | No | Whether the group must be present in every offering. Defaults to `true`. |
| `min_count` | integer | No | Minimum number of items the offering must provide |
| `max_count` | integer | No | Maximum number of items the offering may provide |
| `asset_requirements` | object | No | Technical requirements for each item — use the matching type-specific schema (e.g., `max_length` for text, `min_width`/`aspect_ratio` for images) |

### Product Selectors

**Schema URL**: [`/schemas/core/promoted-products.json`](https://adcontextprotocol.org/schemas/v2/core/promoted-products.json)

Instead of defining explicit offerings, select products from the brand's product catalog:

```typescript
interface ProductSelectors {
  manifest_skus?: string[];      // Specific SKUs
  manifest_tags?: string[];      // Products with these tags
  manifest_category?: string;    // Products in this category
  manifest_query?: string;       // Natural language query
}
```

Multiple selectors are combined with OR logic—products matching ANY criteria are selected.

```json
{
  "brand": {
    "domain": "brand.com"
  },
  "product_selectors": {
    "manifest_tags": ["organic", "seasonal"],
    "manifest_category": "beverages"
  }
}
```

### Asset Selectors

Filter which creative assets from the brand to use:

```typescript
interface AssetSelectors {
  tags?: string[];           // Include assets with these tags
  asset_types?: AssetType[]; // Filter by type (image, video, etc.)
  exclude_tags?: string[];   // Exclude assets with these tags
}
```

```json
{
  "brand": {
    "domain": "brand.com"
  },
  "asset_selectors": {
    "tags": ["holiday", "premium"],
    "asset_types": ["image", "video"],
    "exclude_tags": ["archived"]
  }
}
```

## Use Cases

### Universal format (asset pool)

The buyer provides multiple offerings, each with their own creative asset pool. The publisher picks the most relevant offering and assembles the best headline/image combination:

```json
{
  "brand": { "domain": "acme.com" },
  "offerings": [
    {
      "offering_id": "summer-sale",
      "name": "Summer Sale",
      "landing_url": "https://acme.com/summer",
      "assets": [
        {"asset_group_id": "headlines", "asset_type": "text", "items": [
          {"content": "Shop the Summer Sale"},
          {"content": "50% Off Everything"}
        ]},
        {"asset_group_id": "images_landscape", "asset_type": "image", "items": [
          {"url": "https://cdn.acme.com/summer-hero.jpg", "width": 1200, "height": 628}
        ]}
      ]
    },
    {
      "offering_id": "new-arrivals",
      "name": "New Arrivals",
      "landing_url": "https://acme.com/new",
      "assets": [
        {"asset_group_id": "headlines", "asset_type": "text", "items": [
          {"content": "Just Arrived"},
          {"content": "New This Week"}
        ]},
        {"asset_group_id": "images_landscape", "asset_type": "image", "items": [
          {"url": "https://cdn.acme.com/new-arrivals.jpg", "width": 1200, "height": 628}
        ]}
      ]
    }
  ]
}
```

### Pre-built creatives with brand context

For traditional format types (display banners, video), use `portfolio_ref` to point to pre-built creative variants rather than providing an asset pool:

```json
{
  "brand": { "domain": "brand.com" },
  "offerings": [
    {
      "offering_id": "summer-campaign",
      "name": "Summer Sale",
      "portfolio_ref": "https://brand.com/portfolios/summer-2025.json",
      "landing_url": "https://brand.com/summer"
    }
  ]
}
```

### Conversational-only

No pre-built creatives—just an SI agent for interactive experiences:

```json
{
  "brand": { "domain": "saas-company.com" },
  "si_agent_url": "https://agent.saas-company.com/mcp",
  "offerings": [
    {
      "offering_id": "enterprise-demo",
      "name": "Enterprise Demo",
      "description": "See our platform in action with a personalized demo",
      "keywords": ["demo", "enterprise", "trial", "pricing"]
    },
    {
      "offering_id": "free-trial",
      "name": "14-Day Free Trial",
      "checkout_url": "https://saas-company.com/signup",
      "keywords": ["trial", "free", "signup"]
    }
  ]
}
```

### Multiple time-limited offers with SI

```json
{
  "brand": { "domain": "airline.com" },
  "si_agent_url": "https://agent.airline.com/mcp",
  "offerings": [
    {
      "offering_id": "flash-sale-LAX",
      "name": "Flash Sale: LA Flights",
      "description": "Flights to LA from $99",
      "valid_from": "2025-02-01T00:00:00Z",
      "valid_to": "2025-02-07T23:59:59Z",
      "keywords": ["los angeles", "LA", "cheap flights", "sale"]
    },
    {
      "offering_id": "loyalty-bonus",
      "name": "Double Miles February",
      "description": "Earn 2x miles on all bookings",
      "valid_from": "2025-02-01T00:00:00Z",
      "valid_to": "2025-02-28T23:59:59Z",
      "keywords": ["miles", "loyalty", "rewards", "points"]
    }
  ]
}
```

## Integration with SI Sessions

When `si_agent_url` is present, hosts can initiate SI sessions. The `offering_id` connects the conversation to a specific offering:

### 1. User expresses intent

```
User: "I need flights to LA next week"
```

### 2. Host matches to offering

Host uses `keywords` to find relevant offerings:
- "flash-sale-LAX" matches "LA" keyword
- Check `valid_from`/`valid_to` to confirm offering is active

### 3. Host initiates SI session

```json
{
  "context": "User looking for flights to LA next week",
  "offering_id": "flash-sale-LAX",
  "identity": { "consent_granted": true, "user": { ... } }
}
```

### 4. Brand agent responds with context

```json
{
  "session_id": "sess_abc123",
  "response": {
    "message": "Great timing! Our LA Flash Sale is on now—flights from $99. When are you looking to travel?",
    "ui_elements": [
      {
        "type": "product_card",
        "data": {
          "title": "LA Flash Sale",
          "price": "From $99",
          "cta": { "label": "Search Flights", "action": "search" }
        }
      }
    ]
  }
}
```

## Best Practices

### 1. Match `asset_group_id` to the format's vocabulary

Read the format definition from `list_creative_formats` before building offerings. The `asset_group_id` values must match exactly what the format declares in its `offering_asset_constraints`. A format that requires `images_landscape` will not match a group named `landscape_image`.

### 2. Provide more assets than the minimum

Formats that use asset pools select the best-performing combination. Providing the maximum allowed items gives the publisher more to work with:

```json
{
  "asset_group_id": "headlines",
  "asset_type": "text",
  "items": [
    {"content": "Shop the Summer Sale"},
    {"content": "50% Off Everything"},
    {"content": "Summer Savings Event"},
    {"content": "Limited Time Offer"},
    {"content": "Free Shipping This Week"}
  ]
}
```

### 3. Use `assets` for universal formats, `portfolio_ref` for traditional formats

- **`assets`**: provide when the format assembles the creative from your asset pool (universal/PMax-style formats)
- **`portfolio_ref`**: provide when the creative is pre-built and the publisher picks a matching size/duration

These can coexist on the same offering if you're running across both format types.

### 4. Use keywords for SI discovery

Include relevant keywords so hosts can match user intent to offerings:

```json
{
  "offering_id": "carbon-measurement",
  "name": "Carbon Measurement Platform",
  "keywords": ["carbon", "sustainability", "emissions", "ESG", "scope 3"]
}
```

### 5. Set validity windows

For time-limited promotions, always set `valid_from` and `valid_to`. Hosts filter expired offerings:

```json
{
  "offering_id": "black-friday",
  "name": "Black Friday Sale",
  "valid_from": "2025-11-28T00:00:00Z",
  "valid_to": "2025-11-29T23:59:59Z"
}
```

### 6. Provide both destination URLs when applicable

Include both `checkout_url` (for direct conversion) and `landing_url` (for information):

```json
{
  "offering_id": "premium-plan",
  "checkout_url": "https://app.com/checkout/premium",
  "landing_url": "https://app.com/pricing"
}
```
