Reorganize agent roles to a single source of truth with two depth modes.

`.agents/roles/` is now the canonical source for all subagent prompts. Roles
have two forms:

- `{name}.md` — short triage checker (terse, PR-bound). Used by the triage
  routine's expert consultation step.
- `{name}-deep.md` — long-form design advisor (full domain reasoning) for
  open-ended design work like MCP tool surface review, threat modeling, or
  curriculum architecture. Opt-in; not used in triage.

Nine roles now have a `-deep` counterpart that was previously unreachable —
Claude Code's loader had been silently picking the short `.claude/agents/`
version for the colliding names, leaving the original "agentic API design"
/ "context window efficiency" / "MCP tool design" review specialties on the
shelf for ~7 months. Those are now reachable as `code-reviewer-deep`,
`security-reviewer-deep`, etc.

`.claude/agents/` and `.codex/agents/` are now fully generated by
`scripts/import-claude-agents.mjs`. Don't hand-edit them — edit
`.agents/roles/` and re-run the script.

Addie's expert-panel system-prompt section (`server/src/addie/rules/index.ts`)
filters out `-deep` variants so each persona appears once.
