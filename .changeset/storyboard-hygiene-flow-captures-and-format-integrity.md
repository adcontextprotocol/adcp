Storyboard hygiene follow-ups to PRs #2593 and #2595. Migrates flow-derived media buy IDs to the `$context.<name>` pattern, fixes two context-echo authoring bugs, and adds a `format_id` integrity step to `media_buy_seller`.

**#2585 — flow-derived IDs via `context_outputs:` + `$context.<name>`.** Seven storyboards still hardcoded `mb_*` literals (`mb_acme_q2_2026`, `mb_acme_q2_2026_auction`, `mb_acme_q2_2026_guaranteed`, `mb_acme_q2_2026_proposal`, `mb_nova_q4_2026`, `mb_amsterdam_spring_2026`) against media buys created earlier in the same run. Each `create_media_buy` step now declares `context_outputs: [{ name: media_buy_id, path: "media_buy_id" }]`, and downstream `get_media_buys` / `update_media_buy` / `get_media_buy_delivery` / `report_plan_outcome` steps resolve `"$context.media_buy_id"`. Storyboards covered: `protocols/media-buy/index.yaml`, `protocols/governance/index.yaml`, `specialisms/sales-guaranteed`, `sales-non-guaranteed`, `sales-proposal-mode`, `sales-broadcast-tv`, `sales-catalog-driven`, `creative-generative/generative-seller`. `creative-ad-server` and `governance-delivery-monitor` keep their literals — those storyboards reference prerequisite media buys (Pattern A / `fixtures:` block) and have no creating step to capture from.

**#2589 — context-echo authoring bugs.** Two storyboards asserted `context.correlation_id` on the response but did not send `context:` on the `sample_request`, relying on runner-synthesized context that the spec explicitly forbids (#2593). `content-standards/index.yaml:calibrate_after_policy_change` now sends `context.correlation_id: "content_standards--calibrate_after_policy_change"` (the assertion value was also corrected — it previously asserted against the prior step's correlation_id, so agents that were "passing" by echoing the stale prior-step id will now correctly fail; this is spec-aligned behavior, not a new requirement). `media-buy/scenarios/governance_denied_recovery.yaml:create_media_buy_denied` now sends `context:` explicitly and tightens `field_present: context` to a `field_value: context.correlation_id` echo assertion, exercising the echo-on-error contract from #2589 directly.

**#2597 — `format_id` integrity step on `media_buy_seller`.** New `list_formats_integrity` step in the `product_discovery` phase, between `get_products_brief` and `create_buy`. Filters `list_creative_formats` by `format_ids: ["$context.product_format_id"]` (captured from `get_products`) and asserts `formats[0]` is present. A stale or typo'd `format_ids[]` entry returned by the sales agent now surfaces at discovery time rather than silently at `sync_creatives`. Implements the issue's Option B (baseline check that runs for every seller agent) rather than Option A (opt-in standalone storyboard). Cross-agent dispatch — calling the creative agent declared in `format_ids[].agent_url` directly rather than the sales agent under test — is a follow-up (runner work).
