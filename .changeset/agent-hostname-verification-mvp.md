Enforce agent-hostname ownership at registration time. (Only `organization_domains.verified = true` rows count as a claim — `organizations.email_domain` is not trusted because it can be written from an unverified WorkOS domain via the brand-claim issue flow.) The agent URL's hostname must be exact-or-subdomain of an `organization_domains` row marked `verified = true` for the registering org. Catches the escalation-#340 failure mode (Adzymic registering `adcp-mcp.celtra.com`) at the registration boundary. Gates four write surfaces: `POST /api/me/agents`, `save_agent` MCP tool, `PUT /api/me/member-profile` (bulk profile update — entries already in the JSONB are grandfathered, new URLs go through the gate), `POST /api/me/member-profile` (create), and the visibility-flip routes (`POST/DELETE /agents/:index/publish`, `PATCH /agents/:index/visibility`) when target is `members_only` or `public` — a grandfathered row can't be promoted out of private without satisfying the gate. Orgs with no verified domains fall back to `organizations.email_domain`: corporate signup domains work as soft claims; free-email-provider domains (gmail.com etc.) are rejected outright, closing the personal-workspace attacker vector flagged in the security review. MVP of #4499 — adagents.json delegation and DNS challenge paths for legitimate cross-domain registration follow in later phases. Callers blocked today should add the agent's domain to their organization's Linked Domains (Settings → Organization → Domains) or have an AAO admin register on their behalf via the cross-org admin endpoint from #4498.
