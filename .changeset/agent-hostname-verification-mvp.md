Enforce agent-hostname ownership at registration time. `POST /api/me/agents` and the `save_agent` MCP tool now reject when an org tries to register an agent on a domain it doesn't own — that is, when the agent URL's hostname isn't exact-or-subdomain of any `organization_domains` row marked `verified = true` for the registering org. Catches the escalation-#340 failure mode (Adzymic registering `adcp-mcp.celtra.com`) at the registration boundary instead of relying on after-the-fact admin removal. Orgs with no verified domains (personal workspaces on free email providers) pass through unchanged — there's no claim to enforce against. Existing entries are grandfathered; only new POSTs go through the gate. MVP of #4499 — adagents.json delegation and DNS challenge paths for legitimate cross-domain registration follow in later phases. Callers blocked today should add the agent's domain to their organization's Linked Domains (Settings → Organization → Domains) or have an AAO admin register on their behalf via the cross-org admin endpoint from #4498.
