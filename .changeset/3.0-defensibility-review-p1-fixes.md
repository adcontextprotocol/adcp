Defensibility review P1 fixes (follow-on to the P0 batch in d38468ba2):

- `docs/faq.mdx` — "Do I need to change my measurement stack?" reworded. Old text said "No, AdCP supports standard measurement integrations" — reads as seamless interoperability and conflicts with `known-limitations.mdx` ("not an attribution protocol", not MRC-accredited). New text: "AdCP does not specify attribution or viewability — it is not an MRC-accredited measurement standard. The protocol carries the delivery and usage data your existing IAS/DV/Nielsen/Comscore tools consume."
- `docs/governance/annex-iii-obligations.mdx` — moved the "AdCP is a building block, not a compliance shortcut" framing **before** the "What AdCP provides" obligation table. EU regulators print this page first; the disclaimer needs to be visible before the mechanism table reads as a compliance kit.
- `docs/ai-disclosure.mdx` — replaced vague "Target turnaround is five business days" language for certification grading appeals with a concrete SLA: 72h acknowledgement, 10 business days for human-reviewer decision, outcome (credential granted or fee refunded if upheld), monthly committee escalation, annual transparency report on appeals volume and upheld rate. AAO is both the issuer and the grader of paid certifications; this closes the FTC-style "no published appeal SLA" exposure.
- `docs/reference/known-limitations.mdx` — added "No consent-scope propagation across protocol boundaries" as an explicit 3.0 non-goal, with pointer to the new 3.1 tracking issue (#2540). NOYB-class privacy advocates will read upstream-consent language as implied scope coverage; saying so plainly removes the attack surface.
